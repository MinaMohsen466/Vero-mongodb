# نظام المحاسبة - دليل المستخدم

## 📁 أين يتم حفظ البيانات والاتصال بقاعدة البيانات؟

يعتمد التطبيق على قاعدة بيانات **MongoDB** بدلاً من SQLite. يتم تحديد رابط الاتصال بقاعدة البيانات عبر متغير البيئة `MONGODB_URI` في ملف الإعدادات `.env`.

### كيف يتم تحميل ملف الإعدادات `.env`؟
يبحث التطبيق تلقائيًا عن ملف الإعدادات `.env` عند التشغيل بالترتيب التالي:
1. **بجانب ملف التشغيل المترجم (`.exe`)**: يتيح ذلك إمكانية وضع ملف `.env` بجانب أيقونة تشغيل البرنامج لتغيير رابط قاعدة البيانات في بيئة التشغيل الفعلي (Production) بسهولة ودون الحاجة لإعادة بناء التطبيق.
2. **داخل حزمة التطبيق المدمجة (`app.asar`)**: يتم تضمين ملف `.env` الافتراضي تلقائيًا داخل الحزمة أثناء عملية البناء والتجميع (`npm run build`).
3. **في مجلد المشروع الحالي (`process.cwd()`)**: ويتم استخدامه أثناء بيئة التطوير (Development).

## 🔧 كيف يعمل التطبيق؟

### البنية التقنية

```
┌─────────────────────────────────────────────────────────────┐
│                    واجهة المستخدم (React)                    │
│      الصفحات: لوحة التحكم، الفواتير، العملاء، الإعدادات...   │
└─────────────────────────────────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                  الجسر (Electron IPC)                       │
│         preload.js يربط React مع الـ Main Process          │
└─────────────────────────────────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                 العملية الرئيسية (main.js)                  │
│          يستقبل الطلبات وينفذها على قاعدة البيانات          │
└─────────────────────────────────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                  قاعدة البيانات (MongoDB)                   │
│          (محلياً أو سحابياً عبر MongoDB Atlas)              │
│   المجموعات: customers, suppliers, products, invoices...   │
└─────────────────────────────────────────────────────────────┘
```

### تدفق البيانات

1. **المستخدم** → يضغط زر أو يملأ نموذج
2. **React** → يستدعي `window.api.xxx.yyy()`
3. **Preload** → يرسل للـ main process عبر IPC
4. **Main.js** → ينفذ الاستعلام على قاعدة البيانات من خلال ملف `db.js`
5. **MongoDB** → يحفظ/يقرأ البيانات سحابياً أو محلياً بحسب الرابط المرفق بملف `.env`
6. **النتيجة** ← تعود للواجهة وتُعرض للمستخدم

## 🗄️ جداول قاعدة البيانات

| الجدول | الوصف |
|--------|-------|
| `customers` | بيانات العملاء وأرصدتهم |
| `suppliers` | بيانات الموردين وأرصدتهم |
| `products` | المنتجات والمخزون |
| `invoices` | الفواتير (مبيعات/مشتريات) |
| `invoice_items` | تفاصيل أصناف كل فاتورة |
| `vouchers` | سندات القبض والصرف |
| `accounts` | الحسابات (بنكية/نقدية) |
| `categories` | تصنيفات المنتجات |
| `users` | المستخدمين والصلاحيات |
| `settings` | إعدادات النظام |

## 🔐 النسخ الاحتياطي

### إنشاء نسخة احتياطية
1. اذهب إلى **الإعدادات** → **النسخ الاحتياطي**
2. اضغط **إنشاء نسخة احتياطية**
3. اختر مكان الحفظ

### استعادة نسخة احتياطية
1. اذهب إلى **الإعدادات** → **النسخ الاحتياطي**
2. اضغط **استعادة**
3. اختر ملف النسخة الاحتياطية (.db)

## ⚙️ الإعدادات

### بيانات الشركة
- اسم الشركة، العنوان، الهاتف
- الشعار (يظهر في الفواتير)
- الرقم الضريبي

### إعدادات الفاتورة
- عنوان الفاتورة
- ملاحظة أسفل الفاتورة
- **الشروط والأحكام** (تظهر في طباعة الفاتورة)

### إعدادات عامة
- العملة ورمزها
- عدد الخانات العشرية
- اللغة (عربي/English)
- السماح بالبيع بدون رصيد

## 🖨️ طباعة الفواتير

1. اضغط على أيقونة **الطباعة** 🖨️ بجانب الفاتورة
2. تظهر **معاينة الفاتورة** داخل التطبيق
3. راجع الفاتورة ثم اضغط **طباعة**

### محتويات الفاتورة المطبوعة:
- شعار وبيانات الشركة
- معلومات العميل/المورد
- جدول الأصناف
- المجموع والإجمالي
- حالة الدفع
- الشروط والأحكام (إن وجدت)
- ملاحظة الختام

## ❓ الأسئلة الشائعة

### الفاتورة القديمة لا تظهر أصنافها عند التعديل؟
الفواتير التي تم إنشاؤها في نسخ سابقة قد لا تحتوي على أصناف محفوظة بشكل صحيح.
**الحل:** احذف الفاتورة وأنشئها من جديد.

### أين توجد قاعدة البيانات؟
بما أن التطبيق يستخدم **MongoDB**، يتم تخزين البيانات بشكل افتراضي سحابياً (إذا تم استخدام الرابط السحابي MongoDB Atlas في ملف `.env`). في حال استخدام خادم محلي، يتم حفظها داخل مجلد بيانات خادم MongoDB على جهازك.

### كيف أنقل البيانات لجهاز آخر؟
لا تحتاج لنقل ملفات قاعدة البيانات يدوياً! طالما أن كلا الجهازين يستخدمان نفس رابط الاتصال `MONGODB_URI` في ملف الإعدادات `.env` الخاص بهما، ستتم مزامنة كافة البيانات وتحديثها تلقائيًا على الجهازين في نفس الوقت وبشكل فوري.
